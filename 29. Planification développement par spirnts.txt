# Planification par sprints — TraceForge

Durée d'un sprint : 2 semaines
Objectif : V1 + V2 (portfolio) en ~16 semaines (8 sprints)

---

## Sprint 0 — Setup et architecture (2 semaines)

Objectif : tout ce qu'il faut pour commencer à coder dans de bonnes conditions.

Tâches :
- [ ] Créer le repo GitHub avec la structure monorepo (apps/web, apps/api, docs/)
- [ ] Configurer docker-compose.yml avec tous les services
- [ ] Créer le projet Next.js (App Router, TypeScript strict, Tailwind, Shadcn/ui)
- [ ] Créer le projet FastAPI (structure routers/models/schemas/services/workers/llm/)
- [ ] Configurer ESLint + Prettier (frontend) et Ruff + Black (backend)
- [ ] Écrire les migrations Alembic initiales (toutes les tables du modèle de données)
- [ ] Configurer le pipeline CI GitHub Actions (lint + tests)
- [ ] Écrire le .env.example complet
- [ ] Écrire le README initial avec les instructions de lancement
- [ ] Écrire les 5 ADRs

Livrable : docker-compose up → tous les services up, migrations passent, CI verte

Effort estimé :
- Backend setup : 3 jours
- Frontend setup : 2 jours
- CI + Docker + migrations : 3 jours
- Docs ADR : 1 jour
- Buffer : 1 jour
