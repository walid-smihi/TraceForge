# Architecture Technique — TraceForge

---

## Stack technique

Frontend  : Next.js 14+ (App Router) + TypeScript + Tailwind CSS + Shadcn/ui
Backend   : FastAPI (Python 3.12)
Base de données : PostgreSQL 16 + extension pgvector
Queue     : Redis 7 + RQ (Redis Queue)
Parsing   : PyMuPDF (PDF), python-docx (DOCX), markdown-it
Code analysis : tree-sitter (extraction symboles)
LLM local : Ollama (llama3.1:8b + nomic-embed-text)
LLM cloud : OpenAI / Mistral (optionnel, clé utilisateur)
Déploiement : Docker Compose (local + VPS)
CI        : GitHub Actions

---

## Architecture globale

```
[ Navigateur ]
     |  HTTP / WebSocket
```

## Routes API (FastAPI)

### Projets
GET    /api/v1/projects                       → liste tous les projets
POST   /api/v1/projects                       → crée un projet
GET    /api/v1/projects/{id}                  → détail d'un projet
PUT    /api/v1/projects/{id}                  → modifie un projet
DELETE /api/v1/projects/{id}                  → supprime un projet

### Documents
GET    /api/v1/projects/{id}/documents        → liste les documents
POST   /api/v1/projects/{id}/documents        → upload (multipart/form-data)
GET    /api/v1/projects/{id}/documents/{doc}  → détail d'un document
DELETE /api/v1/projects/{id}/documents/{doc}  → supprime un document

### Exigences
GET    /api/v1/projects/{id}/requirements          → liste (filtres: type, status, priority)
POST   /api/v1/projects/{id}/requirements          → crée manuellement
GET    /api/v1/projects/{id}/requirements/{req}    → détail
PUT    /api/v1/projects/{id}/requirements/{req}    → modifie
DELETE /api/v1/projects/{id}/requirements/{req}    → supprime

### Dépôts de code
POST   /api/v1/projects/{id}/repositories          → importe (local ou GitHub)
GET    /api/v1/projects/{id}/repositories          → liste
GET    /api/v1/projects/{id}/repositories/{repo}/files  → liste des fichiers

### Liens de traçabilité
GET    /api/v1/projects/{id}/trace-links            → liste (filtres: status, source_type)
POST   /api/v1/projects/{id}/trace-links            → crée manuellement
PUT    /api/v1/projects/{id}/trace-links/{link}     → valide / rejette
DELETE /api/v1/projects/{id}/trace-links/{link}     → supprime

### Analyses (tâches asynchrones)
POST   /api/v1/projects/{id}/jobs/extract-requirements   → lance extraction
POST   /api/v1/projects/{id}/jobs/scan-repository        → lance scan repo
POST   /api/v1/projects/{id}/jobs/suggest-links          → lance suggestions
POST   /api/v1/projects/{id}/jobs/impact/{req_id}        → analyse d'impact
GET    /api/v1/jobs/{job_id}                             → statut d'un job (polling)

### Export
GET    /api/v1/projects/{id}/export/impact/{req_id}?format=md   → rapport Markdown
GET    /api/v1/projects/{id}/export/impact/{req_id}?format=pdf  → rapport PDF

### Paramètres LLM
GET    /api/v1/settings/llm        → récupère la config
PUT    /api/v1/settings/llm        → sauvegarde la config
POST   /api/v1/settings/llm/test   → teste la connexion

### Métriques projet
GET    /api/v1/projects/{id}/metrics  → couverture, conflits, qualité

## Flux d'une analyse asynchrone

1. UI : POST /api/v1/projects/{id}/jobs/extract-requirements
2. FastAPI : crée un enregistrement AnalysisJob (status=pending), enqueue dans RQ, retourne {job_id}
3. UI : poll GET /api/v1/jobs/{job_id} toutes les 2 secondes
4. Worker RQ : exécute le job, met à jour job.progress et job.status
5. UI : affiche la progression (42%) → quand status=completed, refresh les données
6. En cas d'erreur : status=failed, error_message visible dans l'UI

## Gestion des embeddings

- Modèle local : nomic-embed-text via Ollama (dim=768)
- Modèle OpenAI : text-embedding-3-small (dim=1536)
- Stockage : colonne `embedding vector(1536)` dans PostgreSQL via pgvector
- Recherche : cosine similarity (<=> opérateur pgvector)
- Threshold de suggestion : similarité > 0.65 → suggéré, > 0.80 → confiance haute

Requête pgvector type :
```sql
SELECT id, path, summary,
       1 - (embedding <=> $1::vector) AS similarity
FROM code_files
WHERE project_id = $2
ORDER BY embedding <=> $1::vector
LIMIT 20;
```
