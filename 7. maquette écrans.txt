# Maquettes Écrans — TraceForge MVP

Pour chaque écran : ce que l'utilisateur voit | actions disponibles | données affichées | états d'erreur / vide / chargement.

---

## Écran 1 — Dashboard projets

URL : /

Ce que l'utilisateur voit :
- Header : logo TraceForge + lien vers les paramètres IA
- Titre : "Mes projets"
- Bouton principal : "Nouveau projet"
- Grille de cartes projets (2-3 colonnes)

Chaque carte projet affiche :
- Nom du projet
- Description (tronquée à 2 lignes)
- Domaine métier (badge : e-commerce, fintech, etc.)
- Métriques : X exigences | Y fichiers | Z liens | W conflits
- Date de dernière modification
- Menu contextuel (3 points) : Ouvrir, Supprimer

Actions :
- Clic sur une carte → ouvre le projet (Écran 2)
- Bouton "Nouveau projet" → modal de création (nom, description, domaine)
- Suppression → confirmation "Supprimer le projet X ? Cette action est irréversible."

État vide :
- Illustration + "Vous n'avez pas encore de projet."
- Bouton "Créer mon premier projet"
- Lien "Ou charger le projet de démo DemoShop"

## Écran 2 — Page projet

URL : /projects/{id}

Ce que l'utilisateur voit :
- Nom du projet en titre + domaine métier
- 4 métriques en haut : Exigences / Fichiers / Liens validés / Conflits détectés
- Barre de progression globale : couverture des exigences (%)
- 5 onglets : Documents | Exigences | Repository | Graphe | Impact

Onglet actif par défaut : Documents si aucun document, sinon Exigences

Actions globales :
- Bouton "Analyser" (relance les suggestions de liens)
- Bouton "Rapport d'impact"

## Écran 3 — Upload documents (onglet Documents)

URL : /projects/{id} → onglet Documents

Ce que l'utilisateur voit :
- Zone de drop (grande, centrée) : "Glissez un fichier PDF, Markdown, TXT ou DOCX"
- Bouton "Parcourir"
- Liste des documents déjà uploadés

Chaque document dans la liste affiche :
- Icône type (PDF/MD/TXT/DOCX)
- Nom du fichier
- Taille
- Statut : En attente | En cours (X%) | Traité | Échoué
- Date d'upload
- Bouton "Supprimer"

Pendant le traitement :
- Barre de progression avec étape visible : "Extraction du texte..." / "Découpage en sections..." / "Extraction des exigences..."

États d'erreur :
- "Fichier trop volumineux (max 50 MB)"
- "Format non supporté. Acceptés : PDF, MD, TXT, DOCX"
- "Erreur lors de l'extraction. Vérifiez que le PDF n'est pas protégé."

## Écran 4 — Liste des exigences (onglet Exigences)

URL : /projects/{id} → onglet Exigences

Ce que l'utilisateur voit :
- Barre de filtres : Type (tous / fonctionnelle / sécurité / performance / ...) | Priorité | Statut | "Sans test" | "Ambiguës"
- Barre de recherche full-text
- Bouton "Ajouter une exigence"
- Tableau des exigences

Colonnes du tableau :
- ID (REQ-001) — trié par défaut
- Titre (tronqué)
- Type — badge coloré
- Priorité — badge (critique=rouge, haute=orange, moyenne=bleu, basse=gris)
- Statut — (draft, actif, modifié, déprécié)
- Liens — icônes : fichiers liés (vert), tests liés (orange), score moyen
- Alerte ambiguïté — icône si détectée
- Actions — icône éditer, icône supprimer

Click sur une ligne → Écran 5 (Détail exigence)
Bouton "Ajouter une exigence" → modal de création

État vide :
- "Aucune exigence extraite. Uploadez un document de spécifications."
- Bouton "Uploader un document"

## Écran 5 — Détail exigence

URL : /projects/{id}/requirements/{req_id}

Ce que l'utilisateur voit :
- Breadcrumb : Projet > Exigences > REQ-001
- Titre de l'exigence (éditable inline)
- Badges : Type | Priorité | Statut
- Alerte ambiguïté (si présente) : "Exigence ambiguë — aucun seuil mesurable détecté. Suggestion : ajouter une latence en ms ou un pourcentage."
- Description complète (éditable)
- Source : nom du document + section d'origine
- Section "Liens de traçabilité" :
  - Sous-section Fichiers liés (avec score et statut de chaque lien)
  - Sous-section Tests liés
  - Bouton "Ajouter un lien manuel"
- Section "Conflits détectés" (si applicable)
- Section "Suggestions non validées" (liens en attente)

Actions :
- Modifier (inline editing sur titre et description)
- Changer type/priorité (dropdowns)
- Valider / Rejeter un lien suggéré
- Ajouter un lien manuel (modal : saisir chemin ou chercher dans la liste)
- "Voir dans le graphe" → Écran 8 centré sur ce nœud
- "Analyser l'impact" → Écran 9 pré-rempli avec cette exigence
- Supprimer l'exigence

## Écran 6 — Import repo (onglet Repository)

URL : /projects/{id} → onglet Repository

Ce que l'utilisateur voit :
- Si pas encore de repo : grande zone d'import avec 2 options

Option A — Local :
- Champ texte : "Chemin absolu vers le dossier" (ex : /home/user/myproject)
- Ou : "Uploader un fichier .zip du repo"

Option B — GitHub :
- Champ texte : "URL du dépôt GitHub public" (ex : https://github.com/user/repo)

- Paramètres avancés (accordéon) :
  - Patterns à exclure (en plus des défauts)
  - Extensions à inclure

- Bouton "Importer et scanner"

Pendant le scan :
- Barre de progression : "Scan des fichiers... 42/156" / "Génération des résumés... 12/156"

Si repo déjà importé :
- Résumé : X fichiers | Y tests | Z langages
- Date du dernier scan
- Bouton "Rescanner"

## Écran 7 — Liste fichiers analysés

URL : /projects/{id} → onglet Repository → liste des fichiers

Ce que l'utilisateur voit :
- Résumé en haut : X fichiers Python | Y fichiers TypeScript | Z tests
- Barre de filtres : Langage | "Tests uniquement" | "Sans lien"
- Barre de recherche (par chemin ou contenu du résumé)
- Tableau des fichiers

Colonnes :
- Chemin (relatif)
- Langage — badge
- Rôle détecté (résumé tronqué)
- Est un test — icône
- Nb liens — avec statut (validés vs suggérés)
- Dernière modif

Click sur un fichier → panneau latéral droit avec :
- Résumé complet
- Entités détectées
- Constantes numériques détectées
- Liste des exigences liées (avec lien vers chacune)

## Écran 8 — Graphe de traçabilité (onglet Graphe)

URL : /projects/{id} → onglet Graphe

Ce que l'utilisateur voit :
- Canvas React Flow pleine largeur
- Nœuds :
  - Exigence : rectangle bleu (REQ-001 — titre tronqué)
  - Fichier source : rectangle vert (src/checkout/reservation.service.ts)
  - Test : rectangle orange (tests/reservation-timeout.test.ts)
  - Ticket : rectangle violet (BUG-421)
- Arêtes :
  - Lien validé : trait plein, flèche, label optionnel
  - Lien suggéré : trait pointillé gris
  - Conflit : trait rouge pointillé avec icône ⚠
- Barre de contrôles (haut gauche) :
  - Filtres : Exigences | Fichiers | Tests | Tickets | Liens validés | Liens suggérés
  - Bouton "Fit view" (recadrer tout)
  - Bouton "Export PNG"
- Mini-map (bas droit)

Click sur un nœud → panneau latéral droit :
- Type et titre du nœud
- Résumé / description
- Liste des nœuds voisins (avec type et lien)
- Bouton "Voir le détail" → redirige vers la fiche complète

État vide :
- "Aucun lien disponible. Importez un document et un dépôt, puis lancez l'analyse."

## Écran 9 — Analyse d'impact (onglet Impact)

URL : /projects/{id} → onglet Impact

Ce que l'utilisateur voit :
- Sélecteur d'exigence (dropdown ou recherche)
- Zone de saisie : "Décrivez la modification envisagée" (texte libre)
- Bouton "Analyser l'impact"

Après analyse :
Section "Exigence modifiée" :
- ID + Titre + ancienne valeur → nouvelle valeur (si applicable)

Section "Fichiers directement impactés" :
- Liste avec chemin, résumé, score de lien

Section "Fichiers indirectement impactés" (distance 2 dans le graphe) :
- Liste avec chemin et explication de la connexion

Section "Tests à rejouer" :
- Liste des tests liés aux fichiers impactés

Section "Conflits détectés" :
- Chaque conflit avec explication lisible

Bouton "Exporter en Markdown"
Bouton "Exporter en PDF" (V2)

## Écran 10 — Paramètres IA

URL : /settings/llm

Ce que l'utilisateur voit :
- Titre : "Configuration du provider IA"
- Sélecteur : Mock | Ollama | OpenAI | Mistral
- Formulaire selon le provider :

  Mock :
  - Aucun paramètre. "Utilisé pour les tests et le développement. Aucun LLM réel."

  Ollama :
  - URL de base : http://ollama:11434
  - Modèle texte : llama3.1:8b (dropdown ou saisie libre)
  - Modèle embeddings : nomic-embed-text
  - Bouton "Tester la connexion"

  OpenAI :
  - Clé API (input masqué, jamais affiché)
  - Modèle : gpt-4o-mini (dropdown)
  - Modèle embeddings : text-embedding-3-small
  - Bouton "Tester la connexion"

  Mistral :
  - Clé API (masquée)
  - Modèle : mistral-small-latest
  - Bouton "Tester la connexion"

- Section "Utilisation ce mois-ci" :
  - Tokens consommés (input + output)
  - Coût estimé total (USD)
  - Détail par opération (extraction, résumé, embeddings, rapport)

Bouton "Sauvegarder"
