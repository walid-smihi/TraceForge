# Environnements — TraceForge

---

## 3 environnements

Local     → développement sur la machine du dev
Staging   → version de test proche de la prod (optionnel pour V1)
Production → version déployée (démo recruteurs)

---

## Local (development)

URL : http://localhost:3000 (frontend) | http://localhost:8000 (API)
Démarrage : docker-compose up --build
DB : PostgreSQL locale dans Docker
LLM : MockProvider (par défaut) ou Ollama local (avec --profile ollama)

Caractéristiques :
- Hot reload activé (frontend et backend)
- Logs verbose
- Swagger UI disponible : http://localhost:8000/docs
- Rate limiting désactivé
- Pas de HTTPS
- Dataset DemoShop chargeable avec make seed-demo

---

## Staging (optionnel, prévu Sprint 7)

URL : https://staging.traceforge.dev
Démarrage : docker-compose -f docker-compose.prod.yml up
DB : PostgreSQL sur VPS (données indépendantes de la prod)
LLM : MockProvider (pas d'Ollama sur staging pour économiser les ressources)

Caractéristiques :
- Identique à la prod techniquement
- Données de test uniquement (peut être wipé à tout moment)
- Dataset DemoShop préchargé
- Déployé automatiquement à chaque merge dans dev (CI)

---

## Production

URL : https://traceforge.dev
Démarrage : docker-compose -f docker-compose.prod.yml up
DB : PostgreSQL sur VPS avec backups quotidiens
LLM : Ollama ou API cloud (configurable)

Caractéristiques :
- HTTPS via Let's Encrypt (Certbot ou Caddy)
- Logs structurés JSON
- Rate limiting activé
- Monitoring : Sentry pour les erreurs
- Dataset DemoShop préchargé
- Déploiement manuel ou sur tag vX.Y.Z

---

## .env.example (à copier en .env)

```bash
# ============================================================
# TraceForge — Configuration
# Copier ce fichier en .env et adapter les valeurs
# ============================================================

# --- Base de données ---
DATABASE_URL=postgresql+asyncpg://traceforge:traceforgedev@postgres:5432/traceforge
POSTGRES_PASSWORD=traceforgedev

# --- Redis ---
REDIS_URL=redis://redis:6379/0

# --- Stockage fichiers ---
STORAGE_PATH=/app/storage

# --- Sécurité ---
# Minimum 32 caractères, changer en production
JWT_SECRET=change-this-in-production-use-a-real-random-secret

# --- LLM Configuration ---
# Valeurs : ollama | openai | mistral | mock
LLM_PROVIDER=ollama

# Ollama (local)
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=llama3.1:8b
OLLAMA_EMBED_MODEL=nomic-embed-text

# OpenAI (optionnel)
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4o-mini

# Mistral (optionnel)
MISTRAL_API_KEY=
MISTRAL_MODEL=mistral-small-latest

# --- Application ---
# Valeurs : development | staging | production | test
APP_ENV=development

# CORS : URL(s) du frontend autorisées (séparées par virgule)
CORS_ORIGINS=http://localhost:3000

# Taille max fichier uploadé en MB
MAX_UPLOAD_SIZE_MB=50
```

## Variables d'environnement frontend (Next.js)

Fichier apps/web/.env.local :
```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
```

En production (dans docker-compose.prod.yml) :
```yaml
environment:
  - NEXT_PUBLIC_API_URL=https://api.traceforge.dev
```

---

## Différences par environnement

| Paramètre | Local | Staging | Production |
|---|---|---|---|
| APP_ENV | development | staging | production |
| LLM_PROVIDER | mock ou ollama | mock | ollama ou openai |
| Rate limiting | désactivé | activé | activé |
| HTTPS | non | oui | oui |
| Logs | console (verbose) | JSON structuré | JSON structuré |
| Sentry | non | non | oui |
| Hot reload | oui | non | non |
| Swagger UI | oui | non | non |
| DemoShop | chargeable | préchargé | préchargé |

## Setup VPS (première fois)

```bash
# Sur le VPS (Ubuntu 22.04 recommandé)
apt update && apt install -y docker.io docker-compose-plugin git

# Créer l'utilisateur app
useradd -m -s /bin/bash traceforge
usermod -aG docker traceforge

# Clone du repo
cd /opt
git clone https://github.com/{username}/traceforge.git
cd traceforge

# Configuration
cp .env.example .env.prod
# Éditer .env.prod avec les vraies valeurs

# Démarrage
docker compose -f docker-compose.prod.yml up -d
docker compose -f docker-compose.prod.yml exec backend alembic upgrade head
docker compose -f docker-compose.prod.yml exec backend python -m scripts.seed_demo

# HTTPS avec Caddy (simple)
apt install -y caddy
# Configurer Caddyfile avec le domaine
```
