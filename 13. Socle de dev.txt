## Commandes Makefile

```makefile
# Démarrage
make up              # docker-compose up --build (avec Ollama)
make up-no-ollama    # docker-compose up sans le profile ollama
make down            # docker-compose down
make logs            # docker-compose logs -f

# Base de données
make migrate         # alembic upgrade head
make migration msg="description"  # créer une nouvelle migration
make db-reset        # supprimer et recréer la DB (attention : supprime les données)

# Développement
make install-web     # npm install dans apps/web
make install-api     # pip install -r requirements-dev.txt dans apps/api

# Tests
make test            # lance tous les tests
make test-api        # pytest apps/api
make test-web        # npm test apps/web

# Qualité code
make lint            # eslint + ruff
make format          # prettier + black
make typecheck       # tsc --noEmit + mypy

# Modèles Ollama
make ollama-pull     # pull llama3.1:8b et nomic-embed-text

# Démo
make seed-demo       # charge le dataset DemoShop dans la DB
```

## Standards backend (Python)

Linter : ruff
Formatter : black (via ruff format)
Type checking : mypy (optionnel, activé progressivement)
Tests : pytest + pytest-asyncio + httpx
Coverage : pytest-cov (objectif : > 80%)

Configuration pyproject.toml :
```toml
[tool.ruff]
line-length = 100
select = ["E", "F", "I", "N", "W"]
ignore = ["E501"]

[tool.ruff.format]
quote-style = "double"

[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["tests"]
```

---

## Standards frontend (TypeScript)

Linter : ESLint (next/core-web-vitals + typescript)
Formatter : Prettier
Type checking : TypeScript strict mode
Tests composants : Vitest + Testing Library
Tests E2E : Playwright (Sprint 7+)

tsconfig.json (strict) :
```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true
  }
}
```

## Variables d'environnement

Copier .env.example → .env au premier setup.

```bash
# Base de données
DATABASE_URL=postgresql+asyncpg://traceforge:traceforgedev@postgres:5432/traceforge

# Redis
REDIS_URL=redis://redis:6379/0

# Stockage fichiers
STORAGE_PATH=/app/storage

# Sécurité (futur usage)
JWT_SECRET=change-this-in-production-min-32-chars

# LLM
LLM_PROVIDER=ollama       # ollama | openai | mistral | mock
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=llama3.1:8b
OLLAMA_EMBED_MODEL=nomic-embed-text

# Cloud LLM (optionnel)
OPENAI_API_KEY=
OPENAI_MODEL=gpt-4o-mini
MISTRAL_API_KEY=
MISTRAL_MODEL=mistral-small-latest

# Application
APP_ENV=development
CORS_ORIGINS=http://localhost:3000
MAX_UPLOAD_SIZE_MB=50
```

## Premier lancement (setup local)

```bash
git clone https://github.com/your-username/traceforge.git
cd traceforge

cp .env.example .env
# éditer .env si besoin

docker-compose up --build

# Dans un autre terminal, lancer les migrations
docker-compose exec backend alembic upgrade head

# Optionnel : charger le dataset de démo
docker-compose exec backend python -m scripts.seed_demo

# Optionnel : télécharger les modèles Ollama
make ollama-pull

# Accès
# Frontend : http://localhost:3000
# API docs : http://localhost:8000/docs
```

## Git hooks (pre-commit)

Fichier .pre-commit-config.yaml :
```yaml
repos:
  - repo: local
    hooks:
      - id: ruff-check
        name: ruff check
        entry: ruff check
        language: system
        types: [python]
      - id: ruff-format
        name: ruff format
        entry: ruff format --check
        language: system
        types: [python]
      - id: eslint
        name: eslint
        entry: npm run lint
        language: system
        files: \.(ts|tsx)$
        pass_filenames: false
```
