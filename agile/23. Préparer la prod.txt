# Préparer la production — TraceForge

Déploiement cible : VPS unique avec Docker Compose (Hetzner CX21 ou équivalent, ~5-10€/mois)

---

## Infrastructure requise

| Composant | Service | Notes |
|---|---|---|
| Backend FastAPI | Docker Compose sur VPS | 2 workers uvicorn |
| Frontend Next.js | Docker Compose sur VPS | Build statique ou SSR |
| PostgreSQL | Docker Compose sur VPS | pgvector inclus |
| Redis | Docker Compose sur VPS | persistence RDB activée |
| Worker RQ | Docker Compose sur VPS | 1 worker |
| Reverse proxy | Caddy ou Nginx | HTTPS automatique |
| Ollama | Optionnel sur VPS | GPU ou CPU lent |
| Stockage fichiers | Volume Docker sur VPS | Migrable vers S3 |

VPS recommandé minimal :
- 4 vCPU, 8 GB RAM (sans Ollama)
- 8 vCPU, 16 GB RAM (avec Ollama llama3.1:8b en CPU)
- 40 GB SSD minimum (pour storage)

---

## Checklist avant premier déploiement

### Infra
- [ ] VPS provisionné et accessible en SSH
- [ ] Docker et Docker Compose installés sur le VPS
- [ ] Domaine DNS configuré (A record → IP VPS)
- [ ] Port 80 et 443 ouverts (firewall)

### Application
- [ ] .env.prod complété avec les vraies valeurs
- [ ] JWT_SECRET aléatoire et fort (min 32 chars)
- [ ] POSTGRES_PASSWORD fort
- [ ] CORS_ORIGINS configuré avec le vrai domaine
- [ ] APP_ENV=production

### Déploiement
- [ ] docker-compose.prod.yml testé localement
- [ ] Migrations Alembic passent en prod
- [ ] Dataset DemoShop chargé
- [ ] Health checks passants

### Sécurité
- [ ] HTTPS actif (Caddy Let's Encrypt ou certificat manuel)
- [ ] Pas de Swagger UI en prod (désactivé si APP_ENV=production)
- [ ] .env.prod non versionné, non exposé
- [ ] Firewall : seuls 80, 443, 22 ouverts

### Monitoring
- [ ] Sentry configuré (SENTRY_DSN dans .env.prod)
- [ ] Logs Docker visibles (docker-compose logs -f)

## Configuration Caddy (HTTPS automatique)

Fichier /etc/caddy/Caddyfile sur le VPS :
```
traceforge.dev {
    reverse_proxy localhost:3000
}

api.traceforge.dev {
    reverse_proxy localhost:8000
}
```

Caddy gère automatiquement Let's Encrypt. Pas besoin de certbot.

## Backups PostgreSQL

Script de backup quotidien (crontab sur VPS) :
```bash
# /opt/traceforge/scripts/backup.sh
#!/bin/bash
BACKUP_DIR=/opt/traceforge/backups
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

docker-compose -f /opt/traceforge/docker-compose.prod.yml exec -T postgres \
  pg_dump -U traceforge traceforge | gzip > $BACKUP_DIR/traceforge_$TIMESTAMP.sql.gz

# Garder seulement les 7 derniers backups
ls -t $BACKUP_DIR/*.sql.gz | tail -n +8 | xargs rm -f
```

Crontab :
```
0 3 * * * /opt/traceforge/scripts/backup.sh >> /var/log/traceforge-backup.log 2>&1
```

## Mise à jour (déploiement d'une nouvelle version)

```bash
# Sur le VPS
cd /opt/traceforge
git pull origin main

docker-compose -f docker-compose.prod.yml up -d --build
docker-compose -f docker-compose.prod.yml exec -T backend alembic upgrade head

# Vérifier que tout est up
docker-compose -f docker-compose.prod.yml ps
curl https://api.traceforge.dev/api/v1/health
```

Rollback si problème :
```bash
git checkout v{previous-tag}
docker-compose -f docker-compose.prod.yml up -d --build
docker-compose -f docker-compose.prod.yml exec -T backend alembic downgrade -1
```
