# Stratégie de coût — TraceForge

---

## Principe

L'app doit fonctionner sans aucune API payante par défaut.
Les providers cloud (OpenAI, Mistral) sont optionnels et nécessitent une clé utilisateur.
Le coût IA est transparent et visible dans l'interface.

---

## Mode par défaut : Ollama local

- Aucun coût API
- Modèle texte : llama3.1:8b (4.7 GB VRAM ou CPU lent)
- Modèle embeddings : nomic-embed-text (274 MB)
- Limitation : latence plus élevée (5-30s par appel selon le hardware)
- Setup : docker-compose --profile ollama up, puis ollama pull llama3.1:8b

---

## Mode optionnel : API cloud (clé utilisateur)

L'utilisateur fournit sa propre clé. TraceForge ne facture rien.

### OpenAI (recommandé pour la qualité)

| Opération | Tokens estimés | Coût estimé |
|---|---|---|
| Extraction exigences (doc 10 pages) | ~8 000 in / ~2 000 out | ~$0.002 |
| Résumé fichier code (~200 lignes) | ~1 500 in / ~300 out | ~$0.0004 |
| Suggestion liens (50 exigences x 30 fichiers) | ~100 000 in / ~15 000 out | ~$0.025 |
| Rapport d'impact | ~3 000 in / ~1 500 out | ~$0.001 |
| Embeddings (200 textes) | ~50 000 tokens | ~$0.001 |

Coût total pour un projet moyen : < $0.05 avec gpt-4o-mini

### Mistral

Comparable à OpenAI en tarif. Bonne alternative pour les projets européens (RGPD).

---

## Mode démo publique

Si l'app est déployée comme démo publique :
- Projet DemoShop préchargé (résultats pré-calculés, pas de nouvelle analyse)
- Analyses désactivées ou limitées (MockProvider)
- Aucun coût LLM pour les visiteurs

## Optimisations coût implémentées

1. Cache des embeddings : si le content_hash d'un fichier n'a pas changé, on ne régénère pas son embedding.
2. Résumés mis en cache : summary stocké en DB, pas recalculé si le fichier n'a pas changé.
3. Batch embeddings : regroupement en batch de 20 (Ollama) ou 100 (OpenAI) pour minimiser les appels API.
4. Skip les petits fichiers : fichiers < 10 lignes ou > 5000 lignes sont gérés différemment (pas de résumé LLM complet).
5. MockProvider en CI/CD : les tests automatisés utilisent toujours MockProvider, zéro coût.
