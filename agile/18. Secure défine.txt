# Stratégie de sécurité — TraceForge

---

## Périmètre V1

V1 = localhost uniquement, pas d'auth, un seul utilisateur.
Les mesures de sécurité V1 se concentrent sur la protection des fichiers et de l'infrastructure.

---

## Gestion des mots de passe et secrets

V1 (pas d'auth) :
- Aucun compte utilisateur, aucun mot de passe à gérer
- Les secrets (clés API) sont stockés en DB hashés ou chiffrés (voir ci-dessous)

V2+ (auth) :
- Hashing : bcrypt avec salt (cost factor 12 minimum)
- Bibliothèque Python : passlib[bcrypt]
- Jamais de mot de passe en clair dans les logs, la DB ou les variables d'environnement

Clés API (OpenAI, Mistral) :
- Option A (V1) : stocker chiffrées en DB avec Fernet (cryptography library)
- Option B : stocker uniquement en session mémoire, non persistées
- Jamais affichées en clair dans l'UI après saisie (input type=password)
- Jamais loggées (masquer dans les logs : "sk-***...")
- Jamais retournées dans les réponses API

---

## Authentification et sessions (V2+)

- JWT (JSON Web Tokens) avec expiration courte (15 min) + refresh token (7 jours)
- Bibliothèque : python-jose
- Stockage côté client : httpOnly cookie (pas localStorage → XSS protection)
- Rotation des refresh tokens à chaque usage
- Invalidation possible (liste noire en Redis)

---

## Validation et sécurité des uploads

Règles implémentées dans le router documents.py :

1. Extensions autorisées : .pdf, .md, .txt, .docx — whitelist stricte
2. Taille maximale : 50 MB (configurable via MAX_UPLOAD_SIZE_MB)
3. Vérification du MIME type réel (pas juste l'extension) via python-magic
4. Stockage hors du dossier public : storage/projects/{id}/documents/
5. Jamais exécuter le contenu uploadé
6. Renommer le fichier à l'upload (UUID + extension) pour éviter path traversal
7. Refuser les noms de fichiers avec : ../, /, \, caractères de contrôle

```python
ALLOWED_EXTENSIONS = {".pdf", ".md", ".txt", ".docx"}
ALLOWED_MIME_TYPES = {
    "application/pdf",
    "text/markdown",
    "text/plain",
    "application/vnd.openxmlformats-officedocument.wordprocessingml.document"
}
MAX_FILE_SIZE = int(os.getenv("MAX_UPLOAD_SIZE_MB", 50)) * 1024 * 1024
```

## Import de dépôt (sécurité)

Pour un dépôt local :
- Valider que le chemin ne sort pas de la zone autorisée (symlink check)
- Limiter la taille totale du repo à 500 MB

Pour un repo GitHub :
- Autoriser uniquement les URLs https://github.com/* (pas d'autres domaines)
- Timeout sur le clone : 120 secondes
- Limiter la taille au clone : max 200 MB

Dans les deux cas :
- Stocker dans storage/projects/{id}/repositories/{repo_id}/
- Ne jamais exécuter de code du repo

## Rate limiting (V1 minimal)

Middleware SlowAPI (fastapi-limiter) :
- Global : 100 req/min par IP
- Upload : 10 uploads/min par IP
- Jobs (analyse) : 5 jobs/min par IP

En V1 local, le rate limiting est désactivé (APP_ENV=development).

## Injection SQL

Protection via SQLAlchemy ORM uniquement (pas de requêtes SQL brutes en string).
Si besoin de requête brute (pgvector), utiliser text() avec paramètres bindés :
```python
from sqlalchemy import text
stmt = text("SELECT * FROM code_files WHERE embedding <=> :vec LIMIT 10")
result = await session.execute(stmt, {"vec": str(embedding)})
```

---

## Injections dans les prompts LLM (Prompt Injection)

- Ne jamais injecter directement du contenu utilisateur dans un prompt sans limite de taille
- Tronquer les textes longs à 4000 tokens max avant envoi au LLM
- Ajouter un rappel système en début de prompt pour ancrer le comportement
- Ne jamais exécuter de code retourné par le LLM
- Les réponses LLM sont parsées en JSON structuré, pas interprétées

---

## Secrets dans le code

- Interdire hardcoding de clés dans le code (ruff rule S105, S106)
- Toutes les valeurs sensibles passent par les variables d'environnement
- .env est dans .gitignore
- .env.example ne contient que des valeurs exemples non fonctionnelles
- Pre-commit hook pour détecter les patterns de clés API dans le code

---

## Suppression des données

L'utilisateur peut supprimer un projet → suppression en cascade :
- Documents + fichiers physiques dans storage/
- Exigences
- Fichiers de code scannés
- Liens de traçabilité
- Jobs et logs LLM

Implémentation : ON DELETE CASCADE en base + nettoyage du dossier storage/ dans le service.
