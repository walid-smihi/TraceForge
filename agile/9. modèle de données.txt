# Modèle de données — TraceForge

ORM : SQLAlchemy (async)
Migration : Alembic
Extension PostgreSQL : pgvector

---

## Entités et schémas complets

### Project

```sql
CREATE TABLE projects (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        VARCHAR(255) NOT NULL UNIQUE,
  description TEXT,
  domain      VARCHAR(100),          -- e-commerce, fintech, santé, industrie, autre
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);
```

### Document

```sql
CREATE TABLE documents (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id      UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  name            VARCHAR(255) NOT NULL,
  file_type       VARCHAR(10) NOT NULL,   -- pdf, md, txt, docx
  file_path       TEXT NOT NULL,          -- chemin dans storage/
  file_size_bytes INTEGER,
  status          VARCHAR(20) DEFAULT 'uploaded',
                  -- uploaded | processing | processed | failed
  error_message   TEXT,
  created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### DocumentChunk

```sql
CREATE TABLE document_chunks (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  document_id   UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  chunk_index   INTEGER NOT NULL,
  content       TEXT NOT NULL,
  section_title VARCHAR(255),
  embedding     vector(768),     -- nomic-embed-text dim=768 / OpenAI dim=1536
  created_at    TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX ON document_chunks USING ivfflat (embedding vector_cosine_ops);
```

### Requirement

```sql
CREATE TABLE requirements (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id        UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  code              VARCHAR(50) NOT NULL,   -- REQ-001, REQ-SEC-002, etc.
  title             VARCHAR(500) NOT NULL,
  description       TEXT,
  req_type          VARCHAR(50) NOT NULL DEFAULT 'functional',
                    -- functional | security | performance | availability | compliance | interface
  priority          VARCHAR(20) DEFAULT 'medium',
                    -- critical | high | medium | low
  status            VARCHAR(20) DEFAULT 'draft',
                    -- draft | active | modified | deprecated
  is_ambiguous      BOOLEAN DEFAULT FALSE,
  ambiguity_reason  TEXT,
  source_document_id UUID REFERENCES documents(id),
  source_chunk_id   UUID REFERENCES document_chunks(id),
  embedding         vector(768),
  version           INTEGER DEFAULT 1,
  created_at        TIMESTAMPTZ DEFAULT NOW(),
  updated_at        TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (project_id, code)
);
CREATE INDEX ON requirements USING ivfflat (embedding vector_cosine_ops);
CREATE INDEX ON requirements (project_id, status);
CREATE INDEX ON requirements (project_id, req_type);
```

### CodeRepository

```sql
CREATE TABLE code_repositories (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id   UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  name         VARCHAR(255) NOT NULL,
  source_type  VARCHAR(20) NOT NULL,   -- local | github
  source_url   TEXT,                   -- URL GitHub si applicable
  local_path   TEXT,                   -- chemin storage/ du repo extrait
  status       VARCHAR(20) DEFAULT 'pending',
               -- pending | scanning | scanned | failed
  scanned_at   TIMESTAMPTZ,
  file_count   INTEGER DEFAULT 0,
  test_count   INTEGER DEFAULT 0,
  created_at   TIMESTAMPTZ DEFAULT NOW()
);
```

### CodeFile

```sql
CREATE TABLE code_files (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  repository_id   UUID NOT NULL REFERENCES code_repositories(id) ON DELETE CASCADE,
  project_id      UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  path            TEXT NOT NULL,          -- chemin relatif dans le repo
  language        VARCHAR(50),            -- python, typescript, javascript, java, go, markdown
  content_hash    VARCHAR(64),            -- SHA-256 pour détecter les changements
  summary         TEXT,                   -- résumé LLM
  role_detected   TEXT,                   -- "gestion de la réservation d'articles"
  entities        TEXT[],                 -- ["article", "stock", "checkout"]
  constants       JSONB,                  -- {"RESERVATION_TIMEOUT_MINUTES": 15}
  is_test_file    BOOLEAN DEFAULT FALSE,
  line_count      INTEGER,
  embedding       vector(768),
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  updated_at      TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (repository_id, path)
);
CREATE INDEX ON code_files USING ivfflat (embedding vector_cosine_ops);
CREATE INDEX ON code_files (project_id, is_test_file);
```

### Ticket

```sql
CREATE TABLE tickets (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id  UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  external_id VARCHAR(100),        -- JIRA-123, BUG-421, GH-456
  source      VARCHAR(20),         -- jira | github | gitlab | csv | manual
  title       VARCHAR(500) NOT NULL,
  description TEXT,
  ticket_type VARCHAR(50),         -- bug | feature | task | story
  status      VARCHAR(100),
  url         TEXT,
  embedding   vector(768),
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

### TraceLink

```sql
CREATE TABLE trace_links (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id       UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  source_type      VARCHAR(20) NOT NULL,   -- requirement | ticket
  source_id        UUID NOT NULL,
  target_type      VARCHAR(20) NOT NULL,   -- code_file | ticket | requirement
  target_id        UUID NOT NULL,
  link_type        VARCHAR(30) DEFAULT 'implements',
                   -- implements | tests | related_to | impacts
  confidence_score DECIMAL(5,2),           -- 0.00 à 100.00
  status           VARCHAR(20) DEFAULT 'suggested',
                   -- suggested | validated | rejected | needs_review
  validation_note  TEXT,
  is_manual        BOOLEAN DEFAULT FALSE,
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  updated_at       TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (source_type, source_id, target_type, target_id)
);
CREATE INDEX ON trace_links (project_id, status);
CREATE INDEX ON trace_links (source_type, source_id);
CREATE INDEX ON trace_links (target_type, target_id);
```

### AnalysisJob

```sql
CREATE TABLE analysis_jobs (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id    UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  job_type      VARCHAR(50) NOT NULL,
                -- extract_requirements | scan_repository | suggest_links | generate_impact
  status        VARCHAR(20) DEFAULT 'pending',
                -- pending | running | completed | failed | cancelled
  progress      INTEGER DEFAULT 0,   -- 0 à 100
  input_data    JSONB,                -- paramètres d'entrée du job
  result_data   JSONB,                -- résultat résumé (nb exigences extraites, etc.)
  error_message TEXT,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  started_at    TIMESTAMPTZ,
  completed_at  TIMESTAMPTZ
);
CREATE INDEX ON analysis_jobs (project_id, status);
```

### LLMUsage

```sql
CREATE TABLE llm_usage (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id          UUID REFERENCES projects(id) ON DELETE SET NULL,
  job_id              UUID REFERENCES analysis_jobs(id) ON DELETE SET NULL,
  provider            VARCHAR(20) NOT NULL,   -- ollama | openai | mistral | mock
  model               VARCHAR(100) NOT NULL,
  operation           VARCHAR(50) NOT NULL,
                      -- extract_requirements | summarize_file | suggest_links
                      -- detect_ambiguity | generate_impact | generate_embeddings
  input_tokens        INTEGER DEFAULT 0,
  output_tokens       INTEGER DEFAULT 0,
  duration_ms         INTEGER,
  estimated_cost_usd  DECIMAL(10,6) DEFAULT 0,
  created_at          TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX ON llm_usage (project_id, created_at);
```

### DetectedConflict

```sql
CREATE TABLE detected_conflicts (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id       UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  rule_id          VARCHAR(20),             -- RULE-002, RULE-004, etc.
  severity         VARCHAR(10) DEFAULT 'warning',   -- critical | warning | info
  title            VARCHAR(500) NOT NULL,
  description      TEXT,
  requirement_ids  UUID[],                  -- exigences impliquées
  status           VARCHAR(20) DEFAULT 'open',
                   -- open | acknowledged | resolved | false_positive
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  resolved_at      TIMESTAMPTZ
);
CREATE INDEX ON detected_conflicts (project_id, status);
```

## Diagram des relations principales

```
Project
  ├── Document (1-N)
  │     └── DocumentChunk (1-N)
  ├── Requirement (1-N)  ←── source: DocumentChunk
  ├── CodeRepository (1-N)
  │     └── CodeFile (1-N)
  ├── Ticket (1-N)
  ├── TraceLink (1-N)   ←── relie Requirement ↔ CodeFile | Ticket | Requirement
  ├── DetectedConflict (1-N)
  └── AnalysisJob (1-N)
        └── LLMUsage (1-N)
```

## Note sur TraceLink

TraceLink est le cœur du modèle. Il permet de relier n'importe quelle entité à n'importe quelle autre :

- requirement → code_file   (l'exigence est implémentée dans ce fichier)
- requirement → code_file   (fichier de test qui valide l'exigence)
- requirement → ticket       (un bug ou feature liée à cette exigence)
- ticket → code_file         (le bug est localisé dans ce fichier)

Le champ `link_type` + `source_type` + `target_type` donnent le sens de la relation.
