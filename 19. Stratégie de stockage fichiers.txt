# Stratégie de stockage fichiers — TraceForge

---

## Principe

V1 : stockage local sur le filesystem (volume Docker)
V2+ : abstraction via StorageService → migration vers S3/MinIO sans changer le code métier

---

## Structure du dossier storage/

```
storage/
  projects/
    {project_id}/
      documents/
        {document_id}.pdf
        {document_id}.md
        ...
      repositories/
        {repo_id}/
          source/           # fichiers du repo cloné ou extrait du zip
            src/
            tests/
            ...
      exports/
        impact_report_{req_id}_{timestamp}.md
        impact_report_{req_id}_{timestamp}.pdf
```

Notes :
- `{project_id}`, `{document_id}`, `{repo_id}` sont des UUIDs
- Les fichiers uploadés sont renommés en UUID pour éviter les conflits et path traversal
- Le nom original est conservé dans la table `documents.name`
- storage/ est monté comme volume Docker et est dans .gitignore

## Interface StorageService (abstraction)

```python
from abc import ABC, abstractmethod
from pathlib import Path

class StorageService(ABC):

    @abstractmethod
    async def save_document(
        self, project_id: str, document_id: str, content: bytes, extension: str
    ) -> str:
        """Sauvegarde un fichier uploadé. Retourne le path relatif."""
        ...

    @abstractmethod
    async def get_document(self, file_path: str) -> bytes:
        """Lit le contenu d'un fichier."""
        ...

    @abstractmethod
    async def delete_document(self, file_path: str) -> None:
        """Supprime un fichier."""
        ...

    @abstractmethod
    async def save_repo(
        self, project_id: str, repo_id: str, zip_content: bytes
    ) -> str:
        """Extrait et stocke un repo depuis un zip. Retourne le chemin."""
        ...

    @abstractmethod
    async def list_repo_files(
        self, repo_path: str, extensions: list[str]
    ) -> list[str]:
        """Liste les fichiers d'un repo avec filtrage par extension."""
        ...

    @abstractmethod
    async def get_file_content(self, file_path: str) -> str:
        """Lit le contenu d'un fichier de code (text)."""
        ...

    @abstractmethod
    async def save_export(
        self, project_id: str, filename: str, content: str
    ) -> str:
        """Sauvegarde un export (rapport Markdown). Retourne le path."""
        ...

    @abstractmethod
    async def delete_project_storage(self, project_id: str) -> None:
        """Supprime tout le dossier storage d'un projet."""
        ...
```

## Implémentation V1 : LocalStorageService

```python
class LocalStorageService(StorageService):

    def __init__(self, base_path: str = "/app/storage"):
        self.base = Path(base_path)

    async def save_document(self, project_id, document_id, content, extension):
        path = self.base / "projects" / project_id / "documents" / f"{document_id}{extension}"
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_bytes(content)
        return str(path.relative_to(self.base))

    async def delete_project_storage(self, project_id):
        import shutil
        project_dir = self.base / "projects" / project_id
        if project_dir.exists():
            shutil.rmtree(project_dir)
```

## Injection dans FastAPI

```python
# config.py
def get_storage_service() -> StorageService:
    if settings.STORAGE_BACKEND == "s3":
        return S3StorageService(bucket=settings.S3_BUCKET)
    return LocalStorageService(base_path=settings.STORAGE_PATH)
```

## Limites et règles

- Taille max par fichier uploadé : 50 MB (configurable MAX_UPLOAD_SIZE_MB)
- Taille max par repo (zip) : 200 MB
- Taille max totale par projet : pas de limite en V1
- Nettoyage des repos après scan : les fichiers source sont gardés pour permettre un rescan
- Les exports sont générés à la demande, non persistés longtemps (suppression après 7 jours en V2)

---

## Migration vers S3 (quand ?)

Migrer vers S3/MinIO quand :
- L'app est déployée sur plusieurs instances (scaling horizontal)
- Le VPS manque de disque
- On veut des backups automatiques

La migration ne touche pas au code métier grâce à l'abstraction StorageService.
Il suffit de changer STORAGE_BACKEND=s3 et de configurer les credentials S3.
