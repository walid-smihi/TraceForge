# Vision Produit — TraceForge

---

## Pitch en une phrase

TraceForge est un outil local-first qui relie automatiquement les exigences d'un projet au code, aux tests et aux tickets, et qui alerte l'équipe sur les incohérences et les impacts lors d'un changement.

---

## Le problème

Dans la plupart des projets logiciels, il n'existe aucune connexion formelle entre :
- les documents de spécification (cahier des charges, user stories, JIRA)
- les fichiers de code qui implémentent ces spécifications
- les tests qui les valident
- les tickets qui signalent des bugs ou évolutions

Résultat : quand une exigence change (un timeout, une règle métier, une contrainte de sécurité), personne ne sait exactement quels fichiers, quels tests, quels tickets sont impactés. Le risque de régression est réel et sous-estimé.

Exemple concret :
> Un développeur réduit le timeout de réservation de 15 minutes à 4 minutes pour optimiser le stock.
> Il ne sait pas que la validation 3D Secure peut prendre jusqu'à 5 minutes.
> Le bug arrive en production : les clients perdent leur panier pendant le paiement.

---

## Ce que c'est

TraceForge est une application local-first qui :
1. Importe des documents de spécification (PDF, Markdown, DOCX)
2. En extrait automatiquement les exigences (via IA)
3. Importe un dépôt de code
4. Analyse les fichiers et génère des résumés
5. Propose des liens entre exigences et fichiers/tests (avec score de confiance)
6. Affiche un graphe de traçabilité interactif
7. Détecte les incohérences et conflits entre exigences
8. Génère un rapport d'impact quand une exigence est modifiée

---

## Ce que ce n'est pas

- Pas un Jira : TraceForge ne gère pas le cycle de vie des tickets.
- Pas un outil de gestion de projet : pas de planning, pas de Gantt.
- Pas un remplaçant aux tests : il détecte des risques, il ne teste pas le code.
- Pas un chatbot généraliste : il ne lit pas tout le repo à chaque question.
- Pas un outil de CI/CD : il ne déploie rien.
- Pas une solution de documentation : il n'écrit pas la doc, il la relie au code.

---

## Valeur produit

Avant TraceForge :
- "Quels fichiers touchent cette exigence ?" → fouille manuelle dans le code
- Modification de timeout sans connaître les impacts → bug en prod
- Exigence floue ("le système doit être rapide") passée inaperçue
- Revue de code sans contexte exigences
- Tests non liés à des exigences → zones risquées invisibles

Avec TraceForge :
- Graphe exigence → code → tests en 1 clic
- Rapport d'impact généré automatiquement avant de merger
- Détection automatique d'ambiguïté + suggestion de métrique
- Chaque fichier est lié à ses exigences
- Vue "exigences sans test" immédiate avec score de couverture

---

## Positionnement

TraceForge se positionne entre :
- Les outils de requirements management lourds (DOORS, Polarion) → trop complexes, réservés à l'industrie aérospatiale/défense
- Les wikis + JIRA → non structurés, liens maintenus manuellement et toujours obsolètes

TraceForge est léger, local, intelligent et actionnable.

---

## Cible principale (V1/V2)

Un développeur ou tech lead qui veut comprendre quelles exigences sont liées à quels fichiers, tests et risques — sans installer un outil enterprise à 50k€/an.

---

## Vision long terme

Devenir l'outil de traçabilité de référence pour les équipes tech soucieuses de la qualité logicielle, du solo developer jusqu'à l'équipe enterprise dans des secteurs sensibles (défense, aérospatiale, médical, finance).
