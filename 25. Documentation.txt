## Fonctionnalités

- Upload de spécifications (PDF, Markdown, DOCX)
- Extraction automatique des exigences (IA)
- Import de dépôt de code (local ou GitHub)
- Suggestions de liens exigence → code → tests
- Graphe de traçabilité interactif
- Détection de conflits entre exigences
- Rapport d'impact exportable

## Stack technique

| Couche | Technologie |
|---|---|
| Frontend | Next.js 14 + TypeScript + Tailwind + Shadcn/ui |
| Backend | FastAPI (Python 3.12) |
| Base de données | PostgreSQL 16 + pgvector |
| Queue | Redis + RQ |
| LLM local | Ollama (llama3.1:8b) |
| Déploiement | Docker Compose |

\`\`\`bash

cp .env.example .env

# Ouvrir http://localhost:3000
\`\`\`

Puis charger la démo :
\`\`\`bash

## docs/setup.md — Contenu

```markdown
# Installation et configuration

## Prérequis
- Docker >= 24
- Docker Compose >= 2.20
- Git

## Installation rapide

1. Cloner le repo
2. Copier .env.example en .env
3. docker-compose up --build
4. Alembic upgrade head
5. (Optionnel) Charger le dataset DemoShop

## Configuration LLM

### Mode Mock (pas de LLM)
LLM_PROVIDER=mock dans .env

### Mode Ollama (local, recommandé)
1. docker-compose --profile ollama up
2. make ollama-pull
3. LLM_PROVIDER=ollama

### Mode OpenAI
1. OPENAI_API_KEY=sk-... dans .env
2. LLM_PROVIDER=openai

## Développement frontend seul
cd apps/web && npm install && npm run dev

## Développement backend seul
cd apps/api && pip install -r requirements-dev.txt
uvicorn main:app --reload
```

## docs/demo-scenario.md — Contenu

```markdown
# Scénario de démo — DemoShop

## Setup
1. Lancer TraceForge : docker-compose up
2. Ouvrir http://localhost:3000
3. Le projet DemoShop est déjà préchargé

## Déroulé de la démo (10 minutes)

### 1. Montrer le dashboard (1 min)
- Présenter le projet DemoShop
- Montrer les métriques : 6 exigences, 8 fichiers, 9 liens, 1 conflit critique

### 2. Montrer les exigences extraites (2 min)
- Aller dans l'onglet Exigences
- Montrer REQ-STOCK-001 (avec timeout modifié 15→4 min)
- Montrer REQ-PAY-002 (3DS max 5 min)
- Ouvrir le détail de REQ-PAY-001 → "ambiguë" (95% manque)

### 3. Montrer le graphe (2 min)
- Aller dans l'onglet Graphe
- Montrer les connexions exigences → fichiers → tests → tickets
- Filtrer "exigences sans test" → montrer qu'elles ressortent
- Cliquer sur BUG-421 → lié à reservation.service.ts et three-ds.service.ts

### 4. Montrer le conflit (2 min)
- Aller dans Impact
- Sélectionner REQ-STOCK-001
- Lancer l'analyse
- Montrer le conflit critique : "réservation expire avant 3DS"
- Montrer les fichiers impactés et les tests à rejouer

### 5. Exporter le rapport (1 min)
- Cliquer "Exporter en Markdown"
- Montrer le rapport généré

### 6. Montrer les paramètres IA (1 min)
- Aller dans Paramètres IA
- Montrer les providers disponibles (Mock/Ollama/OpenAI)
- Montrer le tracking des tokens et coût

## Points à souligner à un recruteur
- Architecture propre : async, workers, embeddings vectoriels
- IA responsable : l'humain valide toujours les suggestions
- Local-first : données jamais envoyées sans clé utilisateur
- Extensible : StorageService abstrait, LLMProvider abstrait
```
